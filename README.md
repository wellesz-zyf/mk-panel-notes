# mk-panel-notes

## 项目简介

本项目是一个轻量级的 HTML 页面归档仓库，用于收集、整理和发布多个独立的静态页面。这些页面内容不针对特定域名或网站，旨在提供一种简洁的方式保存和展示各种面板类、信息类或工具类的 HTML 文档。

仓库结构简单，适合个人或小团队维护，也可用于托管小型静态页面集合。

## 目录说明

```
mk-panel-notes/
├── pages/          # 存放所有归档的 HTML 页面文件
├── assets/         # 页面所需的静态资源（如 CSS、JS、图片等）
├── index.html      # 可选：仓库首页或导航页面
└── README.md       # 本文件
```

- **pages/**：每个 HTML 文件对应一个独立页面，文件名应体现页面内容或用途。
- **assets/**：按需存放页面引用的外部资源，建议按类型或页面组织子目录。
- **index.html**（可选）：若需提供浏览入口，可在此编写简单的导航或说明页面。

## 页面归档说明

- 所有页面均为纯 HTML 文件，不依赖后端服务。
- 页面之间相互独立，无耦合关系。
- 每个页面应尽量自包含，外部资源建议存放在 `assets/` 目录下。
- 页面内容应符合通用网络信息规范，不包含违法、违规或恶意内容。
- 归档遵循“一次提交，长期保留”的原则，不频繁修改已归档页面。

## 维护说明

- 欢迎提交 Pull Request 添加新的 HTML 页面，或修正已有页面中的明显错误。
- 提交前请确保页面在主流浏览器中可正常显示，且无明显兼容性问题。
- 不建议在页面中嵌入跟踪脚本、广告代码或外部不可控资源。
- 若需删除或替换页面，请在 Issue 中说明理由，由维护者评估后处理。
- 本仓库不提供持续集成或自动化部署，页面更新后需手动访问或通过静态托管服务查看。
